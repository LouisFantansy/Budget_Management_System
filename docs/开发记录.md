# 开发记录

更新时间：2026-05-02 09:52 CST

## 当前里程碑

研发预算管理系统已从初始脚手架推进到可运行 MVP 骨架，覆盖预算编制、批量操作、版本审批、一级总表拉取与明细查看、Dashboard 聚合、模板字段校验、权限范围和前端真实 API 接入。

## 已完成模块

### 1. 组织与权限

- 一级部门：`SS`。
- 二级部门：`Arch`、`PVE`、`PE`、`STE`、`PHE`、`FTE`、`cSSD_FW`、`eSSD_FW`、`Embedded_FW`、`PDT`。
- 提供 session 登录、退出和当前用户 API。
- 支持一级预算角色查看全局数据。
- 支持二级预算角色按授权部门查看和编辑。
- 支持审批角色校验，预算编辑角色不能冒充审批人。

### 2. 预算版本闭环

- Draft 预算版本送审。
- 审批通过生成 Approved Version。
- 审批退回回到 Draft。
- Approved/Submitted 版本禁止修改预算行和月度计划。
- 基于 Approved 版本创建修订 Draft。
- 修订 Draft 可再次送审并审批生成 V2。

### 3. 版本 Diff

- 支持同一预算表下两个版本对比。
- 覆盖新增、删除、字段修改和月度金额变化。
- 前端版本分析页已接入真实 Diff API。

### 4. Dashboard

- 支持 `latest_approved` 和 `current_draft` 版本口径。
- 聚合总金额、预算条目数、部门维度、Category 维度、月度金额和数量。
- 前端预算看板页已接入真实聚合 API。
- 支持按部门聚焦筛选看板数据。
- 支持保存个人/部门/全局看板配置。
- 支持设置默认看板配置并应用已保存视图。
- 部门共享和全局共享看板具备权限边界控制。

### 5. 主数据

- 支持 Category、Project Category、Product Line、Project、Vendor、Region 的增删改查。
- Project 主数据新增时支持绑定 `Project Category` 和 `Product Line`。
- Project 主数据列表支持直接修改关联的 `Project Category` / `Product Line`。
- 历史采购建议价接口可按关键字联想推荐。

### 6. 模板字段

- 支持模板字段 CRUD API。
- 支持预算行 `dynamic_data` 按模板字段校验。
- 校验必填、金额/数字、日期、布尔类型。
- PATCH 动态字段时合并 JSON，避免局部编辑覆盖已有字段。
- 前端模板管理页可查看真实字段并新增字段。
- 前端模板管理页支持字段重命名、切换必填和删除字段。
- 预算编制页可动态展示模板字段列，并支持动态字段行内编辑。

### 7. 导入导出

- 新增 `ImportJob` 导入任务模型，记录导入来源、模式、成功行数、错误行数和错误详情。
- 支持预算 Draft 版本的粘贴式 TSV/CSV 导入。
- 支持 `append` 追加导入和 `replace` 覆盖导入。
- 导入校验覆盖标准字段、主数据映射、动态模板字段、月度数量/金额汇总一致性和总金额公式。
- 支持查看导入错误报告。
- 支持预算版本导出 CSV。
- 支持按当前 Draft 版本下载导入模板 CSV。
- 支持按当前 Draft 版本下载示例数据 CSV，内置预算部门、月度数量金额和动态字段示例值。
- 导出 CSV 与导入模板统一复用同一套表头定义，避免字段漂移。
- 前端预算编制页已接入导入面板、最近导入结果、CSV 导出、导入模板下载和示例数据下载按钮。

### 8. Demo 数据

- `seed_demo` 可重复生成演示数据。
- 演示数据覆盖 Draft、Approved、Submitted、修订 Draft、待审批请求和 Dashboard 数据。
- 已加入自动化测试确保 seed 幂等。

### 9. 一级总表前端明细

- 工作台任务卡支持识别 `primary_consolidated` 预算表，并标记为一级总表。
- 工作台支持从任务直接进入对应 Draft 明细，不再依赖默认 Draft 上下文。
- 预算编制页按当前选中 Draft 过滤条目，避免多个版本条目混看。
- 一级总表 Draft 明细页新增来源部门列，展示条目来自哪个二级部门。
- 一级总表支持条目详情查看区，显示来源预算表、来源版本、动态字段和月度计划。
- 一级总表场景保留一级预算管理员对当前 Draft 的编辑能力，不再被前端误判为锁定。
- 新增条目行号按当前 Draft 单独计算，避免跨版本列表导致行号错乱。

### 10. 预算行批量操作

- 新增 `POST /api/budget-lines/bulk/` 批量接口，支持 `delete`、`duplicate`、`patch` 三类操作。
- 批量复制会保留条目主要字段、动态字段和月度计划，并为新条目分配新的 `line_no`。
- 批量删除仅允许作用于当前 Draft，且必须全部属于同一个版本。
- 批量更新复用单条预算行校验逻辑，支持标准字段、`dynamic_data` 和 `local_comments` 合并更新。
- 前端预算编制页新增多选列和批量维护工具条。
- 支持当前 Draft 下批量复制、批量删除、批量更新 `reason`、`purchase_reason` 和批量备注。

### 11. 通知与待办中心

- 新增 `notifications` 应用，提供个人通知模型、列表、摘要和批量标记已读能力。
- 通知分为 `approval_todo`、`approval_result`、`system` 三类，支持未读/已读状态。
- 预算版本送审后，系统自动给审批人生成待办通知。
- 审批通过或退回后，系统自动给提交人生成结果通知。
- 审批人完成处理后，关联待办通知会自动标记为已读，避免脏未读堆积。
- 前端新增通知中心页面，支持查看最近通知、查看未读数和一键全部标记已读。
- 顶部导航新增通知中心入口和未读数红点提示。
- 工作台新增最近通知区块，形成送审、审批、回执的闭环可见性。
