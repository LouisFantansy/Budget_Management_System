# 开发记录

更新时间：2026-05-02 00:10 CST

## 当前里程碑

研发预算管理系统已从初始脚手架推进到可运行 MVP 骨架，覆盖预算编制、版本审批、Dashboard 聚合、模板字段校验、权限范围和前端真实 API 接入。

## 已完成模块

### 1. 组织与权限

- 一级部门：`SS`。
- 二级部门：`Arch`、`PVE`、`PE`、`STE`、`PHE`、`FTE`、`cSSD_FW`、`eSSD_FW`、`Embedded_FW`、`PDT`。
- 提供 session 登录、退出和当前用户 API。
- 支持一级预算角色查看全局数据。
- 支持二级预算角色按授权部门查看和编辑。
- 支持审批角色校验，预算编辑角色不能冒充审批人。

### 2. 预算版本闭环

- Draft 预算版本送审。
- 审批通过生成 Approved Version。
- 审批退回回到 Draft。
- Approved/Submitted 版本禁止修改预算行和月度计划。
- 基于 Approved 版本创建修订 Draft。
- 修订 Draft 可再次送审并审批生成 V2。

### 3. 版本 Diff

- 支持同一预算表下两个版本对比。
- 覆盖新增、删除、字段修改和月度金额变化。
- 前端版本分析页已接入真实 Diff API。

### 4. Dashboard

- 支持 `latest_approved` 和 `current_draft` 版本口径。
- 聚合总金额、预算条目数、部门维度、Category 维度、月度金额和数量。
- 前端预算看板页已接入真实聚合 API。
- 支持按部门聚焦筛选看板数据。
- 支持保存个人/部门/全局看板配置。
- 支持设置默认看板配置并应用已保存视图。
- 部门共享和全局共享看板具备权限边界控制。

### 5. 主数据

- 支持 Category、Project Category、Product Line、Project、Vendor、Region 的增删改查。
- Project 主数据新增时支持绑定 `Project Category` 和 `Product Line`。
- Project 主数据列表支持直接修改关联的 `Project Category` / `Product Line`。
- 历史采购建议价接口可按关键字联想推荐。

### 6. 模板字段

- 支持模板字段 CRUD API。
- 支持预算行 `dynamic_data` 按模板字段校验。
- 校验必填、金额/数字、日期、布尔类型。
- PATCH 动态字段时合并 JSON，避免局部编辑覆盖已有字段。
- 前端模板管理页可查看真实字段并新增字段。
- 前端模板管理页支持字段重命名、切换必填和删除字段。
- 预算编制页可动态展示模板字段列，并支持动态字段行内编辑。

### 7. Demo 数据

- `seed_demo` 可重复生成演示数据。
- 演示数据覆盖 Draft、Approved、Submitted、修订 Draft、待审批请求和 Dashboard 数据。
- 已加入自动化测试确保 seed 幂等。

## 关键后端 API

- `POST /api/budget-versions/{id}/submit/`
- `POST /api/approval-requests/{id}/approve/`
- `POST /api/approval-requests/{id}/reject/`
- `POST /api/budget-books/{id}/create-revision/`
- `GET /api/budget-versions/{id}/diff/?base_version={base_id}`
- `GET /api/dashboard-configs/budget-overview/?version_context=latest_approved`
- `GET /api/dashboard-configs/budget-overview/?version_context=latest_approved&focus_department_id={department_id}`
- `GET /api/dashboard-configs/`
- `POST /api/dashboard-configs/`
- `GET /api/dashboard-configs/{id}/apply/`
- `GET /api/template-fields/?template={template_id}`
- `POST /api/auth/login/`
- `POST /api/auth/logout/`
- `GET /api/auth/me/`
