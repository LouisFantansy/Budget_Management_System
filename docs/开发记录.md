### M1. 一级总表审批闭环与多节点审批引擎

- 新增 `backend/budgets/approval_flow.py`，统一管理审批阶段、节点解析、通知文案和一级总表识别逻辑。
- 二级部门预算仍保持单节点审批；一级总表升级为“两级顺序审批”：
  - `node=1`：一级预算主管/主办初审
  - `node=2`：一级部门负责人终审
- 审批请求 `dashboard_context` 新增 `approval_stage`，支持区分 `secondary_review`、`primary_review`、`final_review`。
- 审批通过逻辑支持多节点顺序流转：
  - 当前节点全部审批完成后，如存在下一节点，则自动推进 `current_node`
  - 自动给下一节点审批人发待办通知
  - 当前版本保持 `submitted`
- 一级总表终审通过后：
  - 预算版本状态落为 `Final`
  - 一级总表 `BudgetBook.status` 落为 `locked`
  - 预算周期 `BudgetCycle.status` 落为 `locked`
  - 相关二级部门任务状态落为 `final_locked`
- 一级总表初审或终审前的送审阶段：
  - 预算周期状态切到 `reviewing`
  - 被汇总的二级部门任务状态切到 `primary_review`
- 一级总表被退回后：
  - 版本回到 `Draft`
  - 一级总表回到 `drafting`
  - 预算周期回到 `active`
  - 被汇总的二级部门任务状态回到 `pulled_to_primary`
- 一级总表修订 Draft 支持基于 `Approved / Final` 版本创建，不再只允许基于 `Approved`。
- `seed_demo` 新增一级审批角色账号：
  - `primary-reviewer / password`
  - `ss-head / password`
- 前端审批中心新增审批阶段透出，可区分“二级审批 / 一级初审 / 一级终审”。

### 2. 预算版本闭环

- Draft 预算版本送审。
- 审批通过生成 Approved Version。
- 审批退回回到 Draft。
- Approved/Submitted 版本禁止修改预算行和月度计划。
- 基于 Approved 版本创建修订 Draft。
- 修订 Draft 可再次送审并审批生成 V2。

### 3. 版本 Diff

- 支持同一预算表下两个版本对比。
- 覆盖新增、删除、字段修改和月度金额变化。
- 前端版本分析页已接入真实 Diff API。

### 4. Dashboard

- 支持 `latest_approved` 和 `current_draft` 版本口径。
- 聚合总金额、预算条目数、部门维度、Category 维度、项目维度、项目类别、产品线、费用类型以及月度金额和数量。
- 前端预算看板页已接入真实聚合 API。
- 支持按部门聚焦和 OPEX/CAPEX 过滤看板数据。

- 支持从月度、部门、Category、项目、项目类别、产品线、费用类型聚合项下钻到预算条目明细。
- 部门共享和全局共享看板具备权限边界控制。

### 5. 主数据

- 支持 Category、Project Category、Product Line、Project、Vendor、Region 的增删改查。
- Project 主数据新增时支持绑定 `Project Category` 和 `Product Line`。
- Project 主数据列表支持直接修改关联的 `Project Category` / `Product Line`。

### 6. 模板字段

- 支持模板字段 CRUD API。
- 支持预算行 `dynamic_data` 按模板字段校验。
- 校验必填、金额/数字、日期、布尔类型。
- 支持基础公式字段，当前可配置 `unit_price * total_quantity`、`sum_month_qty()`、`sum_month_amount()` 等安全公式。
- 保存公式字段时会在服务端校验语法、依赖字段存在性和非法自引用。
- PATCH 动态字段时合并 JSON，避免局部编辑覆盖已有字段。
- 前端模板管理页可查看真实字段并新增字段。
- 前端模板管理页支持字段重命名、切换必填和删除字段。
- 预算编制页可动态展示模板字段列，并支持动态字段行内编辑。

### 6.1 预算条目服务端自动重算

- 新增基础公式引擎，预算行保存时由服务端自动重算 `total_amount`，不再信任客户端传入金额。
- 月度计划新增或修改后，会自动回写预算行 `total_quantity` / `total_amount` 汇总值。
- 公式字段会基于最新预算行主字段和月度计划自动计算并落入 `dynamic_data`。
- 服务端统一保证“公式字段、预算总额、月度汇总”口径一致，为后续导入和专题需求扩展打底。

### 6.2 公式能力落地到导入与前端

- 导入预算时，公式字段不再要求人工填写，系统会根据导入后的主字段和月度计划自动计算。
- 导入示例 CSV 会自动带出公式字段示例值，避免模板使用者误解公式列需要手填。
- 前端模板管理页新增公式字段配置入口，可选择“公式字段”并录入公式表达式。
- 前端预算编制页对公式字段按只读展示处理，避免用户直接改写服务端计算值。

### 6.3 字段级可见性 / 可编辑性 / 保密字段

- 新增模板字段规则解析层，支持 `visible_rules.visible_to` 与 `editable_rules.editable_by` 两类规则，当前以 `primary` / `secondary` 角色范围落地。
- 支持一级预算管理员在模板管理页直接配置“仅一级可见”“仅一级可编辑”。
- 模板字段与预算模板接口会按当前用户裁剪字段，二级用户看不到一级专属保密字段。
- 预算条目接口返回 `field_permissions` 元数据，前端可区分“可见但只读”和“可编辑”字段。
- 服务端会拒绝二级用户通过 API 直接修改一级专属字段，避免前端绕过。
- 预算月度计划改动触发预算行重算时，仍按当前用户可维护字段做校验，不会被隐藏字段误阻塞。
- 送审前动态字段校验改为按当前送审人的可维护字段执行，二级用户不会因为一级专属字段缺失而无法送审。
- 导出 CSV、导入模板、导入示例会按当前用户字段可见性裁剪列。
- 导入校验会拒绝二级用户写入一级专属字段；一级预算管理员仍可导入该类字段。
- 模板字段已支持维护 `width`、`frozen`、`option_source`、`approval_included`、`dashboard_enabled` 和 `import_aliases`。
- 模板支持同周期 schema 修订和跨周期复制，且新激活版本会自动归档旧激活版本。

- 新增 `POST /api/budget-lines/bulk/` 批量接口，支持 `delete`、`duplicate`、`patch` 三类操作。
- 批量复制会保留条目主要字段、动态字段和月度计划，并为新条目分配新的 `line_no`。
- 批量删除仅允许作用于当前 Draft，且必须全部属于同一个版本。
- 批量更新复用单条预算行校验逻辑，支持标准字段、`dynamic_data` 和 `local_comments` 合并更新。
- 前端预算编制页新增多选列和批量维护工具条。
- 支持当前 Draft 下批量复制、批量删除、批量更新 `reason`、`purchase_reason` 和批量备注。

### 11. 通知与待办中心

- 新增 `notifications` 应用，提供个人通知模型、列表、摘要和批量标记已读能力。
- 通知分为 `approval_todo`、`approval_result`、`system` 三类，支持未读/已读状态。
- 预算版本送审后，系统自动给审批人生成待办通知。
- 审批通过或退回后，系统自动给提交人生成结果通知。
- 审批人完成处理后，关联待办通知会自动标记为已读，避免脏未读堆积。
- 前端新增通知中心页面，支持查看最近通知、查看未读数和一键全部标记已读。
- 顶部导航新增通知中心入口和未读数红点提示。
- 工作台新增最近通知区块，形成送审、审批、回执的闭环可见性。

### 12. 集团分摊上传流

- 新增 `AllocationUpload` 上传任务模型，记录集团分摊导入来源、成功/失败状态和错误详情。
- 新增预算周期级接口，支持下载集团分摊模板和导入集团分摊 TSV/CSV。
- 集团分摊导入按“预算部门”列将每条分摊数据映射到具体二级部门，保留部门级分析能力。
- 导入结果会落到 `group_allocation` 来源类型的 OPEX 预算表，并生成当前 Draft 版本。
- 集团分摊条目标记为 `source_ref_type=group_allocation`，且二级部门不可直接编辑。
- 一级总表拉取 OPEX 时会自动把集团分摊 Draft 一并汇入一级总表。
- `seed_demo` 已加入集团分摊样例，便于本地直接验证来源合并效果。
- 前端工作台新增集团分摊模板下载、粘贴导入和最近上传结果展示。

### 13. 一级总表更新提示

- 新增一级总表同步状态检查逻辑，比较当前一级总表 Draft 中引用的二级来源版本与各部门最新 Approved 版本。
- 当某二级部门产生新的 Approved 版本而一级总表仍引用旧版本时，接口会返回待同步部门列表。
- 前端工作台新增一级总表同步提醒卡片，直接提示哪些部门已有新 Approved，建议重新拉取一级总表。
- 提示逻辑仅针对 `primary_consolidated` 预算表，不影响普通部门预算表。

### 14. 专题需求表基础闭环

- 新增 `demands` 应用，提供专题需求模板和专题需求表基础模型。
- 支持一级预算管理员创建专题模板，支持 `secondary` 和 `ss_public` 两种挂载模式。
- 支持创建专题需求表并以 JSON payload 记录专题预算明细。
- 新增 `POST /api/demand-sheets/{id}/generate-budget-lines/`，可将专题需求一键生成到 `special` 来源预算表 Draft。
- 生成条目保留 `source_ref_type=demand_sheet`、`source_ref_id`、来源模板、来源部门和原始行序号追溯信息。
- 生成到预算表中的专题条目统一设置 `editable_by_secondary=False`，二级部门不能在预算编制页直接修改，必须回到专题需求模块维护。
- `SS public` 场景已按业务规则落地为“一级预算管理员直接编制并挂到 `SS public`”，不需要二级部门分摊或代填。
- 前端新增“专题需求”页面，支持查看模板、创建需求表、生成预算和查看最近生成记录。

### 18. 专题需求表配置化与任务化

- 专题模板 `schema` 从“随意 JSON”收敛为有明确语义的字段配置，支持：
  - `text / number / money / date / boolean / option / json` 数据类型
  - `text / number / select / multi_select / date / user / department / project` 输入类型
  - `required / width / frozen / option_source / visible_rules / editable_rules / approval_included / dashboard_enabled / import_aliases`
- 服务端会自动补齐专题模板核心字段：`budget_no`、`description`、`reason`、`unit_price`、`total_quantity`、`total_amount`，并统一校验默认 payload。
- 专题需求表补齐“任务化”字段和状态流转：
  - `draft`：部门填写中，可保存和增删行
  - `submitted`：部门已提交，等待一级确认
  - `confirmed`：一级已确认，可生成预算
  - `generated`：已同步预算，如后续修改则需重新打开后再维护
- 新增专题需求工作流动作：
  - `POST /api/demand-sheets/{id}/submit/`
  - `POST /api/demand-sheets/{id}/confirm/`
  - `POST /api/demand-sheets/{id}/reopen/`
  - 原有 `POST /api/demand-sheets/{id}/generate-budget-lines/` 现在要求需求表已确认后才能执行
- 专题需求表现在会记录：
  - `due_at`
  - `schema_snapshot`
  - `submitted_by / submitted_at`
  - `confirmed_by / confirmed_at`
  - `generated_by / generated_at`
  - `generated_payload_hash`
  - `latest_comment`
- 重新生成预算时继续只重建当前需求表对应的专题条目，不影响同预算表下其他需求表；同时基于 `generated_payload_hash` 暴露“待同步 / 已同步 / 已变更待重同步”状态。
- 前端“专题需求”页改为真实任务台：
  - 左侧按模板创建专题填写任务并指定目标部门/截止时间
  - 中间展示专题任务状态、同步状态和已生成条数
  - 右侧按模板字段做结构化填报，不再以原始 JSON 文本框作为主交互
  - 支持保存、提交、一级确认、重新打开和同步预算
- 字段权限已经贯通到专题需求页：
  - 一级可见/可编辑字段在二级用户下会自动隐藏或只读
  - 二级部门编辑时会保留一级保密字段，不会被覆盖掉

### 19. 主数据扩展与下拉来源治理

- 主数据基础模型扩展为支持 `legacy_name`、`aliases`、`notes`、`keep_history`，满足历史保留和别名治理要求。
- 新增 `CostCenter` 与 `GLAccount` 两类核心主数据：
  - `CostCenter` 可绑定部门
  - `GLAccount` 可绑定费用类型、Category、Project Category
- 新增 `OptionSourceRegistryEntry` 与 `GET /api/option-sources/`，将模板字段下拉来源从“前端约定字符串”提升为“服务端注册目录”。
- 模板字段 `option_source` 现在会经过注册校验：
  - 静态选项继续支持 `A|B|C`
  - 非静态来源必须是系统内置或已注册来源
- 前端主数据页新增 `Cost Center` / `GL Account` 维护能力，并可录入历史名称和别名。
- 预算编制页下拉值解析统一收口到 `resolveOptionValues(...)`，不再在页面里手写各个主数据来源分支。
- 预算导入链路已支持主数据别名命中：
  - `Category / Project / Product Line / Vendor / Region` 等字段除 code/name 外，也可通过 `aliases` 匹配
  - 为后续主数据平滑更名保留兼容能力

### 15. 模块级验证与 GitHub 交付门禁

- 新增 `scripts/verify_module.sh`，按模块统一执行后端检查、模块测试和前端构建。
- 新增 GitHub Actions `CI` 工作流，对 `main` 分支 push 和 PR 执行后端 check、后端测试和前端构建。
- 新增两份交付文档，用于约束“开发一个模块 -> 验证 -> 更新文档 -> 提交 GitHub -> 再进入下个模块”的节奏。

### 16. 一级总表审批回归测试

- 新增一级总表两级审批测试，覆盖：
  - 一级总表送审后生成两节点审批步骤
  - 初审通过后推进到终审节点
  - 终审通过后版本落 `Final`，预算周期与任务状态锁定
  - 初审退回后一级总表与任务状态恢复到待重新处理状态

### 17. 周期分发与任务编排回归测试

- 新增预算周期任务分发测试，覆盖：
  - 一级预算管理员可分发任务并自动创建预算表/Draft
  - 重复分发幂等，不重复创建任务和预算表
  - 二级用户无权执行任务分发
  - 预算任务列表可返回工作台所需真实预算上下文字段
  - 二级用户只能看到自己授权部门任务
