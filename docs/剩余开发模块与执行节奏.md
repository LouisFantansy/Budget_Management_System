# 剩余开发模块与执行节奏

更新时间：2026-05-02 13:05 CST

## 当前真实基线

- 梳理开始时仓库工作树干净，本轮新增的是流程文档、验证脚本和 CI 配置。
- 后端 `manage.py check` 已通过。
- 后端 `manage.py test` 已通过，共 `97` 个测试。
- 前端 `npm run build` 已通过。
- 现阶段已经完成 MVP 主闭环的大部分基础能力：二级预算编制、送审审批、版本 Diff、基础 Dashboard、模板字段、导入导出、批量操作、通知中心、集团分摊、专题需求基础闭环。

## 已完成但仍属 MVP 的模块

- 二级部门预算编制与送审闭环：已可用。
- 一级总表拉取：已可拉取，但尚未形成“一级初审 + 一级终审 + Final 锁定”的完整闭环。
- 模板设计器：已支持字段 CRUD、公式、字段级可见/可编辑，但仍不是完整设计器。
- Dashboard：已支持基础聚合和保存视图，但高级分析、下钻和版本分析能力仍缺。
- 专题需求：已能生成锁定预算条目，但模板配置深度、任务分发和回填同步仍缺。

## 剩余模块优先级

### M1. 一级总表审批闭环与多节点审批引擎

当前状态：
- 角色模型里已有 `PRIMARY_BUDGET_REVIEWER` 和 `PRIMARY_DEPT_HEAD`。
- 审批服务当前只真正覆盖单节点送审闭环。
- 一级总表目前只有“拉取 Draft”，没有“初审 -> 终审 -> Final”闭环。

主要缺口：
- 一级总表 Draft 送一级预算主管初审。
- 初审通过后流转到一级部门负责人终审。
- 终审通过后生成 `Final` 或至少明确的终版锁定状态。
- 审批待办、通知、任务状态、Dashboard 上下文要跟随一级总表流转。

完成标准：
- 一级总表具备完整送审、两级审批、退回、终版锁定能力。
- 审批请求和步骤支持多节点顺序流转，而不是只有 `node=1`。
- `budget_cycles.BudgetTask` 能正确反映 `primary_review` / `final_locked`。

验证命令：
- `scripts/verify_module.sh primary-approval`

### M2. 预算周期与任务分发编排

当前状态：
- `BudgetCycle` / `BudgetTask` 模型已经存在。
- 状态字段已覆盖主要业务状态。
- 目前缺少“按周期配置 -> 分发任务 -> 驱动状态”的完整编排入口。

主要缺口：
- 周期启用、审核中、锁定的状态流转。
- 一级预算管理员按周期批量创建/分发部门任务。
- 任务 owner / due date / 实时状态联动。
- 工作台以任务而不是预算表列表为核心组织。

完成标准：
- 新建预算周期后可一键分发任务到二级部门。
- 二级预算送审、审批通过、一级拉取、一级终审后任务状态自动推进。
- 工作台能够区分“周期状态”和“部门任务状态”。

验证命令：
- `scripts/verify_module.sh cycle-task`

### M3. Dashboard 指标补全与下钻

当前状态：
- 已有 `latest_approved` / `current_draft` 两类上下文。
- 已有部门、Category、月度聚合和保存视图。

主要缺口：
- 缺项目、项目类别、产品线维度聚合。
- 缺 OPEX/CAPEX 分视角、明细表下钻、版本上下文透出。
- 版本分析页仍是基础 Diff 展示，不是真正分析模块。

完成标准：
- Dashboard 至少覆盖需求文档中第一阶段必需的部门、Category、月度、项目、项目类别、产品线聚合。
- 支持从图表定位到预算条目明细。
- 版本上下文在页面和 API 返回中明确可见。

验证命令：
- `scripts/verify_module.sh dashboard`

### M4. 模板设计器深化

当前状态：
- 已支持字段 CRUD、基础公式、字段级权限。
- 导入导出列头已经与模板字段联动。

主要缺口：
- 静态选项 / 主数据来源下拉配置未完整落地。
- 列宽、顺序、冻结列、Dashboard 标签、审批快照标识未落地。
- schema version 与“复制上一周期模板”仍缺自动化。

完成标准：
- 模板字段具备较完整的配置属性。
- 支持模板从上一周期复制，并生成明确 schema version。
- 模板配置变化能稳定影响导入、导出、编制页和 Dashboard 能力。

验证命令：
- `scripts/verify_module.sh template`

### M5. 专题需求表配置化与任务化

当前状态：
- 已有 `DemandTemplate` / `DemandSheet` 基础模型。
- 已支持生成锁定预算条目。

主要缺口：
- 专题模板 schema 还只是 JSON 存储，配置能力不够深。
- 缺填写任务分发、确认、汇总和回填同步。
- 缺“修改专题后重同步预算条目”的清晰操作链路。

完成标准：
- 专题模板支持更完整字段配置。
- 支持按部门分发和跟踪专题填写任务。
- 重新生成预算时能稳定重建本专题关联条目，并保留追溯。

验证命令：
- `scripts/verify_module.sh demands`

### M6. 主数据扩展与下拉来源治理

当前状态：
- 已有 Category / Project Category / Product Line / Project / Vendor / Region。

主要缺口：
- 缺成本中心、GL Account/GL Amount 映射、别名、历史保留、更多主数据实体。
- 模板字段对主数据来源的配置和复用还不完整。

完成标准：
- 核心主数据实体补齐到需求文档第一层。
- 模板字段可以稳定引用主数据源，不再靠前端硬编码假设。

验证命令：
- `scripts/verify_module.sh masterdata`

### M7. 版本分析高级能力

当前状态：
- 已有版本 Diff API 和基础前端展示。

主要缺口：
- 缺版本分支拓扑图。
- 缺修改热力图。

完成标准：
- 可展示某预算表的版本链、修订来源和关键变更统计。
- 页面不再只是静态节点示意图，而是读取真实数据。

验证命令：
- `scripts/verify_module.sh version-analysis`

### M8. 审计、数据血缘与异常提醒

当前状态：
- 目前只有通知和部分来源追溯字段。

主要缺口：
- 缺操作审计日志。
- 缺预算条目生成链路、同步链路的血缘查询。
- 缺异常变更提醒规则。

完成标准：
- 关键操作具备审计记录。
- 一级总表、专题生成、集团分摊能查询来源。
- 具备最小可用的异常变更提醒规则。

验证命令：
- `scripts/verify_module.sh audit`

## 推荐开发顺序

1. `M1 一级总表审批闭环与多节点审批引擎`
2. `M2 预算周期与任务分发编排`
3. `M3 Dashboard 指标补全与下钻`
4. `M4 模板设计器深化`
5. `M5 专题需求表配置化与任务化`
6. `M6 主数据扩展与下拉来源治理`
7. `M7 版本分析高级能力`
8. `M8 审计、数据血缘与异常提醒`

排序理由：
- `M1` 和 `M2` 解决第一阶段闭环还未真正闭上的部分，优先级最高。
- `M3` 直接补齐管理层使用价值。
- `M4` 到 `M6` 是第二阶段配置化能力的主体。
- `M7`、`M8` 属于第三阶段高阶能力，排在后面更稳妥。

## 每模块完成后的固定动作

1. 运行模块级验证命令。
2. 如果本模块改动跨越多个域，再追加执行 `scripts/verify_module.sh full`。
3. 更新 `docs/开发记录.md`、`docs/测试记录.md`。
4. 如果对外说明发生变化，同步 `docs/Wiki草案.md`。
5. 提交并推送到 GitHub 后，再进入下一个模块。
