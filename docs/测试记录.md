## 测试覆盖范围

### 预算审批与版本

- Draft 送审生成审批请求。
- 审批通过生成 V1。
- 审批退回回到 Draft。
- 已审批请求不能再次退回。
- Submitted 版本不能重复送审。
- Submitted/Approved 版本不能修改预算行和月度计划。
- Approved 版本可创建修订 Draft。
- 修订 Draft 可审批生成 V2。

### 版本 Diff

- 新增预算条目识别。
- 删除预算条目识别。
- 字段变化识别。
- 月度计划金额变化识别。
- Diff summary 统计校验。

### Dashboard

- `latest_approved` 默认口径。
- `current_draft` 口径。
- 部门聚合。
- Category 聚合。
- 月度聚合。
- 二级部门用户只能看授权部门。
- 匿名用户返回空业务数据。

### 权限

- 二级用户只能列表查看本部门预算表。
- 一级预算管理员可查看全量预算表。
- 二级用户只能查看本部门审批请求。
- 二级用户不能给其他部门预算表新增预算行。
- 二级用户不能提交其他部门版本。
- 无审批角色用户不能审批请求。

### 模板字段

- 模板字段按 order 返回。
- 重复字段 code 被拒绝。
- 一级预算管理员可创建字段。
- 一级预算管理员可更新字段名称和必填状态。
- 一级预算管理员可删除字段。
- 预算行动态必填字段校验。
- 金额和日期类型错误能定位到具体字段。
- PATCH 动态字段合并已有 JSON。

### Seed 数据

- `seed_demo` 连续运行保持幂等。
- 自动创建一级部门 `SS`。
- 自动创建 10 个二级部门：`Arch`、`PVE`、`PE`、`STE`、`PHE`、`FTE`、`cSSD_FW`、`eSSD_FW`、`Embedded_FW`、`PDT`。
- 自动生成 Draft、Approved、Submitted、修订 Draft 和待审批请求。

## 注意事项

- 从仓库根目录直接运行 `.venv/bin/python backend/manage.py test` 可能无法发现测试。
- 应在 `backend/` 目录运行 `../.venv/bin/python manage.py test`。
- `.venv/`、`node_modules/`、`frontend/dist/`、`backend/db.sqlite3` 不进入版本库。
