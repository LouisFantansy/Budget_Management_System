## 测试覆盖范围

### 预算审批与版本

- Draft 送审生成审批请求。
- 审批通过生成 V1。
- 审批退回回到 Draft。
- 一级总表送审生成两级审批节点。
- 一级总表初审通过后推进到终审节点。
- 一级总表终审通过后生成 `Final` 版本并锁定一级总表。
- 一级总表退回后恢复 `Draft`，并回退预算周期/任务状态。
- 已审批请求不能再次退回。
- Submitted 版本不能重复送审。
- Submitted/Approved 版本不能修改预算行和月度计划。
- Approved 版本可创建修订 Draft。
- Final 版本可创建修订 Draft。
- 修订 Draft 可审批生成 V2。
- Draft 版本支持粘贴式 TSV/CSV 导入。

- 支持导出预算版本 CSV。

## 本轮字段级权限回归关注点

- 二级预算用户读取预算条目时，看不到一级专属保密字段。
- 二级预算用户不能通过 PATCH 修改一级专属字段或一级只读字段。
- 一级预算管理员仍可读取和维护一级专属字段。
- 二级预算用户送审时，不会被一级专属必填字段阻塞。
- 预算导出 CSV、导入模板、导入示例会按当前用户权限裁剪动态字段列。
- 二级预算用户导入包含一级专属字段列时会被拦截，一级预算管理员可正常导入。

## 本轮公式引擎回归关注点

- 预算行创建时会根据 `unit_price * total_quantity` 自动回写 `total_amount`。
- 月度计划新增后，预算行 `total_quantity` / `total_amount` 会自动重算。
- 模板公式字段会基于月度计划和主字段自动计算。
- 非法公式、自引用公式和未知依赖字段会在模板保存阶段被拦截。
- 导入预算时公式字段无需手填，系统会自动计算并写入。
- 导入示例 CSV 会自动生成公式字段示例值。

## 本轮批量操作回归关注点

- 批量接口仅允许作用于同一个 Draft 版本。
- 批量复制会复制条目和月度计划，并生成新的 `line_no`。
- 批量删除只删除选中条目，不影响未选中行。
- 批量更新会复用现有动态字段校验与 JSON 合并逻辑。
- 无权限用户无法对其他部门预算执行批量操作。

## 本轮导入模板回归关注点

- 导出 CSV、导入模板、导入示例三者的表头保持一致。
- 导入模板包含 `预算部门`、12 个月数量列、12 个月金额列和动态模板字段列。
- 示例数据首行可直接作为导入样例使用，满足数量/金额汇总校验。
- 无权限用户无法下载其他部门 Draft 的导入模板和示例。

## 本轮通知中心回归关注点

- 预算送审后，审批人能收到 `approval_todo` 未读通知。
- 审批通过或退回后，提交人能收到 `approval_result` 通知。
- 审批人处理完成后，关联待办通知会自动转为已读。
- 一级总表初审通过后，终审节点审批人会收到新的待办通知。
- 通知摘要未读数与列表数据保持一致。
- 单条标记已读和全部标记已读都能正确更新未读数。

## 本轮一级总表审批回归关注点

- 一级总表送审后 `ApprovalRequest.current_node=1`，且阶段为 `primary_review`。
- 初审通过后请求仍保持 `pending`，并切换到 `current_node=2` / `final_review`。
- 终审通过后一级总表版本变为 `Final`，预算表变为 `locked`。
- 一级总表终审通过后预算周期变为 `locked`。
- 被汇总二级部门任务会按 `pulled_to_primary -> primary_review -> final_locked` 流转。
- 一级总表退回后会恢复为 `Draft`，相关任务回到 `pulled_to_primary`。

## 本轮集团分摊回归关注点

- 一级预算角色可以下载集团分摊模板并成功导入多行分摊数据。
- 导入时按“预算部门”映射到二级部门，未知部门会被准确拦截。
- 集团分摊导入会生成 `group_allocation` 来源预算表和 Draft 版本。
- 一级总表拉取 OPEX 时会合并二级自编 Approved 与集团分摊 Draft。
- `seed_demo` 连续运行仍保持幂等，并稳定生成集团分摊样例。

## 本轮一级总表更新提示回归关注点

- 一级总表刚完成拉取时，同步状态接口返回无更新。
- 某二级部门产生新的 Approved 版本后，同步状态接口会返回待同步部门列表。
- 工作台在存在待同步部门时会展示提醒卡片。

## 本轮专题需求回归关注点

- 一级预算管理员可以创建专题需求任务，完成“提交 -> 一级确认 -> 生成预算”完整链路。
- 二级专题会落到对应二级部门的 `special` 来源预算表 Draft。
- `SS public` 专题只允许挂到 `SS public`，且仅一级预算管理员可维护。
- 专题生成的预算条目保留来源追溯字段和月度计划。
- 同一预算表下多张专题需求表可并存，重建单张需求表不会误删其他专题条目。
- 二级部门用户不能直接修改锁定的专题预算条目。
- 已确认或已同步的专题需求表不能直接编辑，必须先重新打开。
- 一级保密字段对二级用户自动隐藏；二级编辑时不会覆盖一级保密字段原值。
