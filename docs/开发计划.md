# 研发预算管理系统开发计划

## 1. 技术路线

- 后端：Django 4.2 LTS + Django REST Framework。当前本机 Python 为 3.9，Django 4.2 是兼容且稳定的 LTS 版本；后续如果升级 Python 到 3.12，可再评估 Django 5.x。
- 前端：Vue 3 + Vite + TypeScript + Pinia + Vue Router。
- UI 风格：苹果风格的内部管理台，强调轻量玻璃质感、柔和阴影、细边框、留白、清晰层级、低饱和背景和高密度表格能力。
- 数据库：开发阶段默认 SQLite 方便启动；生产目标 PostgreSQL。
- 架构：前后端分离，REST API 通信；后续导入导出、Diff 计算和 Dashboard 聚合接入异步任务。

## 2. 第一阶段目标：预算编制闭环 MVP

第一阶段先跑通“模板配置 -> 部门预算编制 -> 版本送审 -> 审批通过 -> 一级总表拉取 -> Dashboard 展示”的最小闭环。

### 2.1 后端范围

- 账号与角色基础模型。
- 组织、一级/二级部门、成本中心。
- 预算周期和任务状态。
- OPEX/CAPEX 模板、字段、schema version。
- 预算 Book、Draft、Approved Version、预算行、月度计划。
- 二级部门负责人审批流的基础状态机。
- 历史采购名称匹配和推荐价接口。
- 一级总表拉取最新 Approved 版本的基础结构。

### 2.2 前端范围

- 苹果风格应用框架：侧边栏、顶部栏、工作台、状态卡片。
- 预算工作台：预算周期、部门任务、待审批入口。
- 预算编制表页面骨架：后续接入高性能表格。
- Dashboard 页面骨架：默认 Approved 口径，支持版本上下文提示。
- 模板管理页面骨架。

## 3. 第二阶段目标：配置化与专题表单

- 完整模板设计器。
- 字段级权限、保密字段、导入导出映射。
- 专题需求收集表配置、分发、填写、生成预算条目。
- 专题生成条目锁定，二级部门只能回专题表单修改需求。
- Dashboard 自定义配置、保存和分享。

## 4. 第三阶段目标：高级分析与运营能力

- 版本分支拓扑图。
- 修改热力图。
- 预算编制迭代次数分析。
- 类智慧城市预算总览大屏。
- 操作审计、数据血缘和异常变更提醒。

## 5. 当前启动任务

- 初始化 `backend/` Django 项目。
- 初始化 `frontend/` Vue3 项目。
- 建立基础领域模型、API 路由和前端页面结构。
- 跑通后端 `manage.py check` 和前端构建。
